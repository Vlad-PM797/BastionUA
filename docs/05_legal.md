### Рекомендована структура

```

           │  TOV "Bastion UA"       │ ← ТОВ на спрощенці (Україна)
           │  (основна операційна)   │
           └─────────────┬───────────┘
                         │
        ┌────────────────┼────────────────┐
        │                │                │
┌───────▼───────┐ ┌──────▼──────┐ ┌──────▼──────┐
│ ФОП-кофаунд.  │ │ ФОП-команда │ │ ФОП-контракт.│
│ (equity)      │ │ (зарплата)  │ │ (субпідряд) │
└───────────────┘ └─────────────┘ └─────────────┘

           │
           ▼
┌──────────────────────────────────┐
│  Estonian OÜ (optional, Y2)      │ ← для міжнар. IAP збору
│  для Steam / App Store Global    │
└──────────────────────────────────┘
```

**Чому ТОВ, а не ФОП-одиничний**:
- Можливість мати кількох співзасновників з частками
- Можливість прийняти інвестицію (equity)
- Публічна звітність = довіра партнерів-фондів
- Нормальний договір з Apple/Google Developer (потрібна юрособа для бренду)

**Чому дод. Estonian OÜ (для року 2+)**:
- Простота роботи з Stripe/Paddle/RevenueCat
- Низькі податки на нерозподілений прибуток (0% до дивідендів)
- Швидше IBAN в ЄС
- Для УА можуть бути валютні обмеження (НБУ ліміти)

**Альтернатива**: Полька SP.z o.o. або німецька UG (для євроцентричного бізнесу).

## 2. Ліцензії на символіку (бригади)

### 2.1. Загальний принцип

ЗСУ, Нацгвардія, МВС — **державні формування**. Їх символіка (шеврони, назви, офіційні логотипи) формально належать державі, але:

- Багато бригад мають **власні фонди/ГО** з зареєстрованими торговельними марками
- Деякі шеврони є **об'єктами авторського права** конкретного автора-художника
- Кілька бригад (Азов, 3 ОШБр, Червона Калина) — мають активний бренд-менеджмент

### 2.2. Партнерські бригади (приклад)

| **ГУР МО** | Закритий | Крім випадків спец.домовленостей |

### 2.3. Модель ліцензування

**Варіант A — MOU (Memorandum of Understanding)**:
- Безоплатна ліцензія на використання назви/шеврону
- В обмін: % від donation carve-out → фонд бригади + PR в грі
- Термін: 2 роки з автопродовженням
- Сторона може відкликати ліцензію

**Варіант B — Licensing agreement (платне)**:
- Royalty (%) з revenue пов'язаного контенту (Skin-бриґада)
- Flat fee за використання шеврону + revenue share
- Exclusivity can (лише наша гра в ігровій тематиці)
- Складніше, потрібен повноцінний юрист

**Рекомендація для MVP**: Варіант A з 4-5 бригадами-партнерами. Це більш етично і просто.

### 2.4. Template MOU (структура)

```
1. Сторони: TOV "Bastion UA" + [ГО / Фонд / Бригада]
2. Предмет: право використання назви, номера, шеврону у грі
3. Обсяг: мобільна гра + веб + соц.мережі
4. Обмеження:
   - Не використовувати у рекламних матеріалах без схвалення
   - Не продавати мерч з символікою поза грою
   - Зобов'язання: не дискредитувати бренд, не спотворювати факти
5. Фінансові відносини:
   - Безкоштовна базова ліцензія
   - X% (5-15%) від profit attributable to brigade content → фонд бригади
   - Reports: квартальні
6. Термін + припинення
7. IP / моральні права
8. Форс-мажор (війна!)
9. Суперечки: український суд
10. Підписи
```

### 2.5. Державна символіка

**Тризуб, Герб, Державний Прапор, Гімн**:
- Використання регулюється ЗУ "Про державні символи України"
- Декоративне використання в патріотичному ключі **дозволено**
- Заборонено: спотворення, комерційний продаж з символом без згоди, розміщення в принизливих контекстах
- Рекомендація: юр. консультація до релізу + узгодження з Мінкультом

**Прапор ЗСУ, військові символи**:
- ЗУ "Про статут внутрішньої служби" + підзаконні акти
- Для **реалістичного** зображення (як в історичному документі) — ризик нижчий
- Узгодження з Генштабом — опціональне, але бажане

## 3. Реальні особи в грі

### 3.1. Публічні фігури

**Допустиме без окремої згоди**:
- Залужний, Сирський, Буданов, Умєров — в ролі історичних фігур в реалістичному контексті
- Зеленський, Резніков — теж (public figures + історична точність)
- Реалістичний стиль (без карикатур, без спотворення позиції)

**Рекомендація**:
- Отримати попередньо `дозвіл` через прес-службу (усне/письмове) — знижує ризики
- Аудіо/відео з реальними обличчями — **бажано** отримати згоду
- Референси з відкритих джерел (прес-конференції, офіційні виступи) — безпечніше

### 3.2. Воїни / полеглі герої

**Критично важливо**:
- Для будь-якого реального героя → **письмова згода родини** або офіційного фонду
- Memorial Hall = лише з офіційно опублікованою інформацією + додаткова згода для імені в грі
- Ніколи не створювати фіктивних сюжетів з реальними іменами
- Використання лише публічних даних (інтервʼю, офіційні некрологи) — мінімальна згода усно

### 3.3. Колабораціоністи / вороги

**Реальні вороги (путін, Шойгу, Герасимов, Лавров тощо)**:
- Можна використовувати (публічні фігури) в сатиричному / карикатурному контексті
- НЕ робити їх "геймплейними об'єктами" (наприклад, "вбити путіна" — проблематично для App Store)
- Краще: "Ворог", "Окупант", з натяком через сюжет (промови, пропагандистські клипи)

**App Store / Google Play ToS**:
- Apple 1.1.1: "targeted personal attacks" — заборона **прямих** атак на конкретних людей
- Google 10: "hate speech / harassment"
- Рекомендація: загальний ворог ≠ конкретна особа у геймплеї

## 4. App Store / Google Play compliance

### 4.1. Apple App Store

**Risks**:
- 1.1.6: Mature themes (war violence) → Rating 12+/17+
- 1.2: Social Network → User-generated content (клани) → moderation required
- 3.1.1: IAP disclosure compliance
- 4.3: Spam / duplicate content (не наш кейс)
- 5.1.1: Data privacy (GDPR + CCPA compliance)

**Mitigation checklist**:
- [ ] Age rating: **12+** (зображення насильства обмежено, без gore)
- [ ] Жодних графічних смертей, без крові
- [ ] Rewards при донатах → чітке розкриття (IAP vs реальні фонди)
- [ ] Privacy Policy + Terms українською + англійською
- [ ] Data handling: мінімум PII (email, device_id); no tracking without consent
- [ ] Чітка disclosure в App Store listing: "Гра містить теми війни"

### 4.2. Google Play

**Risks**:
- Inappropriate Content — war / violence
- Real World Violence policy
- User Generated Content moderation

**Mitigation**:
- [ ] Rating ESRB Teen (13+) або PEGI 12
- [ ] No real-world harm targeting
- [ ] Moderation pipeline для кланів/чатів (анти-флуд, анти-спам)
- [ ] Donation disclosure: "Партнери-фонди" не "ми збираємо для ЗСУ"

### 4.3. Ін-апп покупки vs донати

**Критична важливість**:
- Apple / Google **забороняють** збір грошей на треті цілі через IAP (окрім COVID-like винятків)
- Solution: **НЕ проводити реальні донати через IAP**. Використовувати **зовнішні линки** в фонди
- IAP обслуговує лише ігрову валюту (Cubes, Battle Pass). Донати = окремий екран з deep-links
- Apple 3.1.1 (IAP) vs 3.2.2 (дозволяє зовнішні посилання на donations for approved nonprofits)

**Для українських фондів**:
- Otrymaty підтвердження "approved nonprofit" через Apple Bridge або Google `non-profit` program (Фонд Притули вже є в списках)
- Якщо ні — просто external link з попередженням "Ви переходите на сторонній сайт"

## 5. GDPR / CCPA / DMA compliance

### Мінімум для European users

- Privacy Policy: зрозуміла, українською + англійською
- Cookie/Tracking consent modal (Usercentrics / Didomi)
- Right to be forgotten (delete account → 30 днів)
- Data Protection Officer (DPO): призначити або outsource
- Data Processing Agreement з Google / Apple / Sentry / AppsFlyer

### Для US (CCPA, COPPA)

- Opt-out tracking
- Age gate 13+ (у нашому випадку 12+ → особлива обережність)
- No targeted ads to minors

### Для UA (ЗУ "Про захист персональних даних")

- Реєстрація бази персональних даних (Омбудсмен)
- Згода на обробку
- Повідомлення про витік (72 год)

## 6. Donation integration (юридично)

### Структура

**Партнер-модель (рекомендована)**:

```
User → [ ВАША ГРА з deep-link ] → [ Сторонній додаток фонду ]
                                   (monobank jar / FundraiseUp / UA24)
```

- Жодні донати не проходять через рахунок розробника
- Гравець отримує in-game бонус (косметичний бейдж) після "підтвердження донату" (soft-verify через screenshot або API)
- Рахунок — безпосередньо у фонд
- Звіт: фонд публікує + ми агрегуємо

**Чому так**:
- Уникнення статусу "фінансового посередника"
- Немає податків на донати (ми не отримувач)
- Немає проблем з Apple 3.1.1
- Прозорість для аудитора

**"20% carve-out net profit"**:
- Це **договірна** сплата **з нашої маржі** (не від revenue)
- Оформлюється як "charitable contribution" / "спонсорський внесок"
- Документи: квартальний звіт + акт виконаних робіт + призначення платежу

### Податкове оптимізування

**УА**: благодійність на ЗСУ = 100% cost deduction (ЗУ "Про благодійну діяльність" + податкові пільги воєнного часу)
**ЕU**: через Estonian OÜ — благодійність вираховується з податку до 10% оборот
**US**: 501(c)(3) status партнера потрібен для deduction

## 7. IP — асети, музика, голос

### 7.1. 3D-моделі

- Synty Studios → ліцензія OK для коммерч.
- Quixel Megascans → через Unreal / Unity ліцензію
- Mixamo → free commercial
- Sketchfab → перевіряти індивідуальну ліцензію

### 7.2. Музика

**Не можна**:
- Олег Винник, Океан Ельзи, Вакарчук, Тарабарова — без ліцензії
- Гімн України — потрібна згода МКСК
- Пісні гурту "Антитіла" / "ТНМК" — перевіряти ліцензії

**Рекомендовано**:
- Композитор-на-контракт (оригінальний саундтрек)
- Ліцензовані бібліотеки: Epidemic Sound, Soundstripe, Artlist
- Український фолк через CC-BY

### 7.3. Голос

- Диктор/актор за контрактом (buy-out rights)
- Реальні промови (Залужний, Зеленський) — через прес-службу, з посиланням на джерело
- Жодного клонування voice через AI без дозволу

### 7.4. Фото / відео

- Getty Images / Shutterstock — комерційна ліцензія
- UkrInform, Укрінформ-Агентство — зверну з ліцензією
- Офіційні канали ЗСУ / Генштабу — за запитом
